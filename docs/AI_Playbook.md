## Merge-Policy

**Alle Feature-Branches werden über Pull Requests im GitHub-Browser gemerged.**
- Keine direkten `git merge main` Aktionen lokal
- PR Review & Merge durch User (Owner)
- KI erstellt PR-Beschreibung + Testhinweise

Vorteile:
- saubere History
- leichter Rollback
- transparente Änderungen
- Review vor Merge

Damit arbeitet die KI wie ein echter Team-Dev.
